NPM সেই Foundational Tool, যেটা দিয়ে সবকিছু শুরু হয়েছিল— **NPM (Node Package Manager)**। এটি বিশ্বের বৃহত্তম সফটওয়্যার রেজিস্ট্রি এবং Node.js ইনস্টল করলে এটি ডিফল্টভাবেই পিসিতে চলে আসে।

নিচে NPM ব্যবহারের পূর্ণাঙ্গ গাইড দেওয়া হলো:

---

### ১. NPM ইনস্টল করা (Installation)

NPM আলাদাভাবে ইনস্টল করতে হয় না। আপনি যদি [nodejs.org](https://nodejs.org/) থেকে **Node.js** ইনস্টল করেন, তবে NPM স্বয়ংক্রিয়ভাবে ইনস্টল হয়ে যাবে।

- **ভার্সন চেক করা:**

```bash
node -v
npm -v

```

---

### ২. নতুন প্রজেক্ট শুরু করা (Initialize)

যেকোনো নতুন ফোল্ডারকে জাভাস্ক্রিপ্ট প্রজেক্ট বানানোর জন্য:

```bash
npm init

```

_(সব অপশন অটোমেটিক সেট করতে চাইলে `npm init -y` ব্যবহার করুন। এতে একটি `package.json` ফাইল তৈরি হবে।)_

---

### ৩. প্যাকেজ ম্যানেজমেন্ট (Core Commands)

NPM-এ প্যাকেজ হ্যান্ডেল করা খুব সহজ:

- **প্যাকেজ ইনস্টল করা:**

```bash
npm install <package-name>
# উদাহরণ: npm install express

```

- **একসাথে সব ডিপেন্ডেন্সি ইনস্টল করা:**
  (যদি আপনার কাছে শুধু `package.json` ফাইল থাকে)

```bash
npm install

```

- **ডেভ-ডিপেন্ডেন্সি হিসেবে সেভ করা:**

```bash
npm install <package-name> --save-dev

```

- **গ্লোবাল ইনস্টল (পুরো কম্পিউটারের জন্য):**

```bash
npm install -g <package-name>

```

- **প্যাকেজ ডিলিট করা:**

```bash
npm uninstall <package-name>

```

---

### ৪. স্ক্রিপ্ট রান করা (Running Scripts)

`package.json` ফাইলের `scripts` সেকশনে থাকা কমান্ডগুলো চালাতে হয় এভাবে:

```bash
npm run <script-name>
# উদাহরণ: npm run dev অথবা npm run start

```

_(নোট: শুধু `npm start` এবং `npm test`-এর জন্য `run` না লিখলেও চলে, তবে অন্য সব ক্ষেত্রে দিতে হয়।)_

---

### ৫. কিছু দরকারি কমান্ড (Pro Tips)

- **প্যাকেজে কোনো সিকিউরিটি সমস্যা আছে কি না চেক করতে:**

```bash
npm audit

```

- **অপ্রয়োজনীয় প্যাকেজ ক্লিন করতে:**

```bash
npm prune

```

- **ক্যাশ মেমোরি ক্লিয়ার করতে:**

```bash
npm cache clean --force

```

### ৪.৫. প্যাকেজ আপডেট করা (Update Packages)

প্রজেক্টের প্যাকেজগুলো পুরনো হয়ে গেলে সেগুলো আপডেট করা জরুরি:

- **কোন প্যাকেজগুলো আপডেট দরকার তা দেখতে:**

```bash
npm outdated

```

- **প্যাকেজ আপডেট করতে:**

```bash
npm update

```

---

### ৫. NPX - ইনস্টল না করেই প্যাকেজ চালানো

এটি NPM-এর একটি অসাধারণ ফিচার। কোনো প্যাকেজ পিসিতে পার্মানেন্টলি ইনস্টল না করে শুধু একবার রান করার জন্য `npx` ব্যবহার করা হয়।

- **উদাহরণ (React প্রজেক্ট তৈরি):**

```bash
npx create-react-app my-app

```

> এটি আপনার হার্ডড্রাইভের জায়গা বাঁচায় কারণ প্যাকেজটি স্থায়ীভাবে ইনস্টল না করেই একবার ব্যবহার করা যায়।

---

### ৭. Semantic Versioning (প্যাকেজ ভার্সন বুঝুন)

`package.json` ফাইলে প্যাকেজের ভার্সন সাধারণত এভাবে থাকে: `^1.2.3`

- **^ (Caret):** প্রথম সংখ্যা ঠিক রেখে বাকিগুলো আপডেট করবে (যেমন: ১.৩.০)। এটি সেফ।
- **~ (Tilde):** শেষের সংখ্যাটি শুধু আপডেট করবে (যেমন: ১.২.৪)।
- **কোনো চিহ্ন না থাকলে:** একদম ফিক্সড ওই ভার্সনটাই থাকবে।

---

### ৮. Lockfile এর গুরুত্ব (`package-lock.json`)

অনেকে এই ফাইলটি ডিলিট করে দেয়, কিন্তু এটি খুব গুরুত্বপূর্ণ।

- এটি নিশ্চিত করে যে আপনার টিমের সবাই এবং সার্ভার—সব জায়গায় একদম **সেম ভার্সন** ইনস্টল হচ্ছে।
- এটি প্রজেক্টের ডিপেন্ডেন্সি ট্রি-এর একটি স্ন্যাপশট ধরে রাখে।

---

### কিছু প্রো-টিপস (Pro Tips - Updated):

- **প্রজেক্টের সব প্যাকেজের লিস্ট দেখতে:**

```bash
npm list --depth=0

```

- **শর্টকাট ব্যবহার করুন:** - `npm install` এর বদলে শুধু `npm i` লিখলেই হয়।
- `npm install -D` মানেই হলো `--save-dev`।
- `npm i -g` মানেই হলো গ্লোবাল ইনস্টল।

### 9. NPM কেন ব্যবহার করবেন?

1. **সবচেয়ে বড় কমিউনিটি:** যেকোনো লাইব্রেরি বা প্যাকেজ সবার আগে NPM-এ আসে।
2. **বিশ্বাসযোগ্যতা:** এটি অনেক পুরনো এবং স্টেবল।
3. **বিল্ট-ইন:** আলাদা করে কোনো টুল ইনস্টল করার ঝামেলা নেই।
