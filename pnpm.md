**pnpm (Performant NPM)** বর্তমানে ডেভেলপারদের অনেক পছন্দের কারণ এটি Bun-এর মতোই দ্রুত এবং এটি কম্পিউটারের স্টোরেজ অনেক সাশ্রয় করে। pnpm মূলত **"content-addressable storage"** ব্যবহার করে, যার ফলে একই প্যাকেজ বারবার ডাউনলোড হয় না।

নিচে pnpm ব্যবহার করার পূর্ণাঙ্গ গাইড দেওয়া হলো:

---

### ১. pnpm ইনস্টল করা (Installation)

আপনার পিসিতে যদি Node.js ইনস্টল করা থাকে, তবে নিচের কমান্ডটি দিয়ে খুব সহজে pnpm ইনস্টল করতে পারেন:

- **npm ব্যবহার করে ইনস্টল (সবচেয়ে সহজ):**

```bash
npm install -g pnpm

```

- **অন্যান্য উপায়:** যদি আপনার পিসিতে Node.js না থাকে, তবে Windows-এ `iwr https://get.pnpm.io/install.ps1 -useb | iex` অথবা Mac/Linux-এ `curl -fsSL https://get.pnpm.io/install.sh | sh -` ব্যবহার করতে পারেন।

ইনস্টল শেষ হলে `pnpm -v` লিখে চেক করে নিন।

---

### ২. প্রজেক্ট শুরু করা (Initialization)

নতুন প্রজেক্টের জন্য `package.json` ফাইল তৈরি করতে ব্যবহার করুন:

```bash
pnpm init

```

---

### ৩. প্যাকেজ ম্যানেজমেন্ট (Common Commands)

NPM-এর সাথে এর কমান্ডের অনেক মিল আছে, শুধু `npm` এর জায়গায় `pnpm` লিখতে হয়:

- **প্যাকেজ ইনস্টল করা:**

```bash
pnpm add <package-name>

```

- **ডেভ-ডিপেন্ডেন্সি হিসেবে সেভ করা:**

```bash
pnpm add -D <package-name>

```

- **সব ডিপেন্ডেন্সি ইনস্টল করা (node_modules তৈরি):**

```bash
pnpm install

```

- **গ্লোবাল প্যাকেজ ইনস্টল করা:**

```bash
pnpm add -g <package-name>

```

- **প্যাকেজ রিমুভ করা:**

```bash
pnpm remove <package-name>

```

---

### ৪. স্ক্রিপ্ট রান করা (Running Scripts)

আপনার `package.json` ফাইলের স্ক্রিপ্টগুলো রান করতে `run` ব্যবহার করুন:

```bash
pnpm run dev
# অথবা
pnpm dev

```

_(pnpm-এর সুবিধা হলো অনেক ক্ষেত্রে `run` না লিখলেও সরাসরি স্ক্রিপ্ট চিনে নেয়)_

---

### ৫. pnpm কেন সেরা? (সুবিধাসমূহ)

1. **স্পেস সাশ্রয়:** pnpm আপনার কম্পিউটারে প্রতিটি প্যাকেজ একবারই সেভ করে। ১০টি প্রজেক্টে যদি `react` ব্যবহার করেন, তবে এটি আপনার হার্ডড্রাইভে একবারই জায়গা নেবে (Hard links ব্যবহার করে)।
2. **দ্রুতগতি:** এটি npm এবং সুপরিচিত yarn-এর চেয়ে অনেক দ্রুত কাজ করে।
3. **Strictness:** npm-এ কোনো প্যাকেজ ইনস্টল না করেও মাঝে মাঝে অ্যাক্সেস করা যায় (Phantom dependencies), কিন্তু pnpm এটি হতে দেয় না। ফলে বাগ হওয়ার সম্ভাবনা কমে যায়।

---

### ৬. NPM থেকে pnpm-এ মাইগ্রেট করবেন কীভাবে?

যদি আপনার পুরনো কোনো প্রজেক্ট থাকে যেখানে `package-lock.json` আছে, তবে সেখানে সরাসরি নিচের কমান্ডটি দিন:

```bash
pnpm import

```

এটি আপনার পুরনো লকফাইল থেকে অটোমেটিক `pnpm-lock.yaml` তৈরি করে ফেলবে। এরপর পুরনো `node_modules` ডিলিট করে `pnpm install` দিন।

---

**একটি জরুরি নোট:** pnpm ব্যবহার করলে এটি `pnpm-lock.yaml` ফাইল তৈরি করবে। প্রজেক্ট শেয়ার করার সময় এটিও গিট-এ পুশ করবেন।
