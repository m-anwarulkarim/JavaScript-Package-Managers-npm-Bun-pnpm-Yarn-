**pnpm (Performant NPM)** বর্তমানে ডেভেলপারদের অনেক পছন্দের কারণ npm ও yarn-এর তুলনায় অনেক দ্রুত এবং এটি কম্পিউটারের স্টোরেজ অনেক সাশ্রয় করে। pnpm মূলত **"content-addressable storage"** ব্যবহার করে, যার ফলে একই প্যাকেজ বারবার ডাউনলোড হয় না।

নিচে pnpm ব্যবহার করার পূর্ণাঙ্গ গাইড দেওয়া হলো:

---

### ১. pnpm ইনস্টল করা (Installation)

আপনার পিসিতে যদি Node.js ইনস্টল করা থাকে, তবে নিচের কমান্ডটি দিয়ে খুব সহজে pnpm ইনস্টল করতে পারেন:

- **npm ব্যবহার করে ইনস্টল (সবচেয়ে সহজ):**

```bash
npm install -g pnpm

```

- **অন্যান্য উপায়:** যদি আপনার পিসিতে Node.js না থাকে, তবে Windows-এ `iwr https://get.pnpm.io/install.ps1 -useb | iex` অথবা Mac/Linux-এ `curl -fsSL https://get.pnpm.io/install.sh | sh -` ব্যবহার করতে পারেন।

ইনস্টল শেষ হলে `pnpm -v` লিখে চেক করে নিন।

---

### ২. প্রজেক্ট শুরু করা (Initialization)

নতুন প্রজেক্টের জন্য `package.json` ফাইল তৈরি করতে ব্যবহার করুন:

```bash
pnpm init

```

---

### ৩. প্যাকেজ ম্যানেজমেন্ট (Common Commands)

NPM-এর সাথে এর কমান্ডের অনেক মিল আছে, শুধু `npm` এর জায়গায় `pnpm` লিখতে হয়:

- **প্যাকেজ ইনস্টল করা:**

```bash
pnpm add <package-name>

```

- **ডেভ-ডিপেন্ডেন্সি হিসেবে সেভ করা:**

```bash
pnpm add -D <package-name>

```

- **সব ডিপেন্ডেন্সি ইনস্টল করা (node_modules তৈরি):**

```bash
pnpm install

```

- **গ্লোবাল প্যাকেজ ইনস্টল করা:**

```bash
pnpm add -g <package-name>

```

- **প্যাকেজ রিমুভ করা:**

```bash
pnpm remove <package-name>

```

---

### ৪. স্ক্রিপ্ট রান করা (Running Scripts)

আপনার `package.json` ফাইলের স্ক্রিপ্টগুলো রান করতে `run` ব্যবহার করুন:

```bash
pnpm run dev
# অথবা
pnpm dev

```

_(pnpm-এর সুবিধা হলো অনেক ক্ষেত্রে `run` না লিখলেও সরাসরি স্ক্রিপ্ট চিনে নেয়)_

---

### ৬. NPM থেকে pnpm-এ মাইগ্রেট করবেন কীভাবে?

যদি আপনার পুরনো কোনো প্রজেক্ট থাকে যেখানে `package-lock.json` আছে, তবে সেখানে সরাসরি নিচের কমান্ডটি দিন:

```bash
pnpm import

```

এটি আপনার পুরনো লকফাইল থেকে অটোমেটিক `pnpm-lock.yaml` তৈরি করে ফেলবে। এরপর পুরনো `node_modules` ডিলিট করে `pnpm install` দিন।

---

**একটি জরুরি নোট:** pnpm ব্যবহার করলে এটি `pnpm-lock.yaml` ফাইল তৈরি করবে। প্রজেক্ট শেয়ার করার সময় এটিও গিট-এ পুশ করবেন।

### ৩.৫. ইন্টারঅ্যাক্টিভ আপডেট (Pro Feature)

pnpm-এর একটি দারুণ ফিচার হলো এর ইন্টারঅ্যাক্টিভ আপডেট মোড। এটি ব্যবহার করলে আপনি টার্মিনালেই লিস্ট দেখতে পাবেন এবং সিলেক্ট করে আপডেট করতে পারবেন।

```bash
pnpm update --interactive

```

> এটি আপনাকে অ্যারো কি (Arrow keys) দিয়ে প্যাকেজ সিলেক্ট করে আপডেট করার সুযোগ দেয়, যা অনেক সহজ।

---

### ৫.১. স্টোরেজ মেকানিজম (কিভাবে জায়গা বাঁচায়?)

সাধারণত NPM প্রতিটি প্রজেক্টে আলাদা করে প্যাকেজ ডাউনলোড করে। কিন্তু pnpm একটি **Global Content-addressable Store** ব্যবহার করে।

- **Hard Links:** আপনার কোড ফোল্ডারে থাকা ফাইলগুলো আসলে অরিজিনাল স্টোরেজের একটি রেফারেন্স মাত্র। ফলে ৫০০টি প্রজেক্টে একই প্যাকেজ থাকলেও সেটি আপনার হার্ডড্রাইভে ১ বারই জায়গা দখল করে।

---

### ৭. মনো-রেপো সাপোর্ট (Workspace)

বর্তমানে বড় প্রজেক্টগুলো (Monorepo) ম্যানেজ করার জন্য pnpm-এর **Workspaces** ফিচারটি সবচেয়ে বেশি জনপ্রিয়। একই ফোল্ডারে অনেকগুলো সাব-প্রজেক্ট থাকলে এটি দিয়ে সেগুলো নিয়ন্ত্রণ করা যায়।

- রুট ডিরেক্টরিতে `pnpm-workspace.yaml` ফাইল তৈরি করে এটি কনফিগার করা হয়।
- সব প্রজেক্টে একসাথে কমান্ড চালাতে:

```bash
pnpm -r run build

```

---

### ৮. ডিরেক্ট স্টোর পাথ দেখা

আপনার পিসিতে pnpm ঠিক কোথায় সব প্যাকেজ জমা করে রাখছে তা দেখতে চাইলে এই কমান্ডটি দিন:

```bash
pnpm store path

```

যদি কখনো মনে হয় স্টোরেজ ক্লিনিং দরকার, তবে ব্যবহার করুন:

```bash
pnpm store prune

```

### pnpm এর কিছু পাওয়ারফুল কমান্ড (একনজরে)

| কমান্ড           | কাজ                                                          |
| ---------------- | ------------------------------------------------------------ |
| `pnpm dlx`       | প্যাকেজ ইনস্টল না করেই রান করা (npx-এর বিকল্প)               |
| `pnpm why <pkg>` | কোনো প্যাকেজ কেন ইনস্টল হয়েছে তার কারণ দেখা                  |
| `pnpm list`      | প্রজেক্টের সব ডিপেন্ডেন্সির লিস্ট দেখা                       |
| `pnpm server`    | একটি ব্যাকগ্রাউন্ড সার্ভার চালানো যা ইনস্টলেশন আরও ফাস্ট করে |

### প্যাকেজ ম্যানেজার বাছাই করার ফাইনাল ডিসিশন :

| আপনার প্রয়োজন হলে...                              | সেরা চয়েস |
| ------------------------------------------------- | --------- |
| **সবচেয়ে দ্রুত ইনস্টলেশন এবং রানটাইম**           | **Bun**   |
| **হার্ডড্রাইভের জায়গা বাঁচানো (Shared Store)**    | **PNPM**  |
| **বড় এন্টারপ্রাইজ প্রজেক্ট ও স্ট্যাবিলিটি**       | **Yarn**  |
| **বেসিক লার্নিং এবং কোনো ঝামেলা ছাড়া ডিফল্ট টুল** | **NPM**   |

**উপসংহার:**
যাঁরা কম স্টোরেজে অনেকগুলো প্রজেক্ট নিয়ে কাজ করতে চান এবং প্রজেক্টের ডিপেন্ডেন্সিগুলোর মধ্যে স্বচ্ছতা (Strictness) বজায় রাখতে চান, তাঁদের জন্য **pnpm** বর্তমানে ইন্ডাস্ট্রির গোল্ড স্ট্যান্ডার্ড।

1. **স্পেস সাশ্রয়:** pnpm আপনার কম্পিউটারে প্রতিটি প্যাকেজ একবারই সেভ করে। ১০টি প্রজেক্টে যদি `react` ব্যবহার করেন, তবে এটি আপনার হার্ডড্রাইভে একবারই জায়গা নেবে (Hard links ব্যবহার করে)।
2. **দ্রুতগতি:** এটি npm এবং সুপরিচিত yarn-এর চেয়ে অনেক দ্রুত কাজ করে।
3. **Strictness:** npm-এ কোনো প্যাকেজ ইনস্টল না করেও মাঝে মাঝে অ্যাক্সেস করা যায় (Phantom dependencies), কিন্তু pnpm এটি হতে দেয় না। ফলে বাগ হওয়ার সম্ভাবনা কমে যায়।
