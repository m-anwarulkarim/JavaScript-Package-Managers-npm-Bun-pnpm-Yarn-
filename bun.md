নিচে **Bun** ইনস্টল করা থেকে শুরু করে প্রজেক্টে এটি ব্যবহারের পূর্ণাঙ্গ প্রসেস দেওয়া হলো:

---

### ১. Bun ইনস্টল করা (Installation)

প্রথমে আপনার কম্পিউটারে Bun ইনস্টল করতে হবে। টার্মিনাল বা কমান্ড প্রম্পট ওপেন করে নিচের কমান্ডটি দিন:

- **macOS এবং Linux-এর জন্য:**

```bash
curl -fsSL https://bun.sh/install | bash

```

- **Windows-এর জন্য:**
  উইন্ডোজে এটি Powershell-এর মাধ্যমে ইনস্টল করা যায়:

```powershell
powershell -c "irm bun.sh/install.ps1 | iex"

```

ইনস্টল হয়ে গেলে `bun --version` লিখে চেক করে দেখুন এটি কাজ করছে কি না।

---

### ২. নতুন প্রজেক্ট শুরু করা (Initialize Project)

NPM-এ আমরা যেমন `npm init` দিতাম, Bun-এ দিতে হবে:

```bash
bun init

```

এটি আপনার প্রজেক্টে `package.json` এবং একটি `index.ts` বা `index.js` ফাইল তৈরি করে দেবে।

---

### ৩. প্যাকেজ ইনস্টল করা (Install Packages)

Bun-এর সবচেয়ে বড় সুবিধা হলো এর স্পিড। এটি NPM-এর চেয়ে ২০-৩০ গুণ দ্রুত প্যাকেজ ইনস্টল করতে পারে।

- **সব প্যাকেজ ইনস্টল করতে (npm install এর বদলে):**

```bash
bun install

```

- **নির্দিষ্ট কোনো প্যাকেজ যোগ করতে (npm install <package> এর বদলে):**

```bash
bun add express

```

- **ডেভ-ডিপেন্ডেন্সি হিসেবে যোগ করতে:**

```bash
bun add tailwindcss -d

```

- **প্যাকেজ রিমুভ করতে:**

```bash
bun remove express

```

---

### ৪. স্ক্রিপ্ট রান করা (Running Scripts)

আপনার `package.json` ফাইলে যদি কোনো স্ক্রিপ্ট থাকে (যেমন: `start` বা `dev`), তবে সেটি রান করতে `run` কমান্ড ব্যবহার করুন:

```bash
bun run start

```

অথবা সরাসরি কোনো ফাইল রান করতে চাইলে:

```bash
bun index.ts

```

### 5. হট রিলোডিং (Hot Reloading)

ডেভেলপমেন্টের সময় ফাইল সেভ করলেই সার্ভার অটোমেটিক রিস্টার্ট হওয়ার জন্য Bun-এ বেশিরভাগ ক্ষেত্রে আলাদা `nodemon` ছাড়াই কাজ চলে যায়।
সরাসরি নিচের কমান্ডটি ব্যবহার করুন:

```bash
bun --watch index.ts

```

> এটি আপনার কোড পরিবর্তন ট্র্যাক করবে এবং মুহূর্তের মধ্যে রিফ্রেশ করবে।

---

### 6. এনভায়রনমেন্ট ভ্যারিয়েবল (.env support)

Bun-এর একটি দুর্দান্ত ফিচার হলো এটি কোনো থার্ড-পার্টি লাইব্রেরি (যেমন: `dotenv`) ছাড়াই `.env` ফাইল পড়তে পারে।

- প্রজেক্টের রুট ডিরেক্টরিতে একটি `.env` ফাইল তৈরি করুন।
- কোডের ভেতরে সরাসরি ব্যবহার করুন: `process.env.PORT` ।

---

### 7. Bun দিয়ে টেস্টিং (Fast Testing)

Bun-এর নিজস্ব বিল্ট-ইন টেস্ট রানার আছে যা Jest-এর সাথে সামঞ্জস্যপূর্ণ কিন্তু অনেক গুণ দ্রুত।

- আপনার টেস্ট ফাইলের নাম দিন `index.test.ts` বা `math.test.ts`।
- রান করতে ব্যবহার করুন:

```bash
bun test

```

---

### 9. NPM থেকে Bun-এ মাইগ্রেশন (Quick Migration)

আপনার যদি আগে থেকে কোনো Node.js প্রজেক্ট থাকে, তবে খুব সহজেই মাইগ্রেট করতে পারেন:

1. পুরাতন `node_modules` ডিলিট করে দিন: `rm -rf node_modules`
2. `bun install` কমান্ড দিন।
3. ব্যাস! এখন থেকে `npm run start`-এর বদলে `bun run start` ব্যবহার করুন।

### 10. কেন Bun ব্যবহার করবেন?

- **Native TypeScript & JSX Support:** কোনো ট্রান্সপাইলার (যেমন: Babel/TSC) ছাড়াই সরাসরি `.ts` বা `.tsx` ফাইল রান করা যায়।
- **Web Standard APIs:** এখানে `fetch`, `WebSocket`, এবং `ReadableStream` বিল্ট-ইন থাকে।
- **Single Binary:** এটি আপনার প্রজেক্টকে একটি সিঙ্গেল বাইনারি ফাইলে রূপান্তর করতে পারে (Bun Build), যা সার্ভারে ডিপ্লয় করা অনেক সহজ।
